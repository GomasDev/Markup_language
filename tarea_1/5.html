formularios

El lenguaje de html no solamente sirve para representar infromacion, tambien es usado para enviar datos a traves de el al servidor o a algun servicio concreto.

Etiqueta form

Tiene unos atributos destacados en base a su funcionabilidad.
Atributo methodDefine la forma de comunicacion con el protocolo por defecto usado (normalmente HTTP) y su valor por defecto es GET(envia los datos en la cabecera del paquete), POST(envia los datos en el
cuerpo del paquete)
Atributo action vamos a poner el esquema y recepor de los datos enviados por el formulario, por defecto sera el protocolo http y la misma pagina donde estoy. Atributo onsubmit es un atributo de evento, es decir, que
podremos definir ejecuciones javascript. Para cortar el envio de datos le daremos el valor "return false;"

Etiquetas input y label

Etiqueta label

Acompañar informacion a los campos de interacion su atributo mas importante for que nos dice a que campo de interacion se aplica mediante el id del campo.

Etiqueta input

campos de datos para el usuario

Sus atributos mas importantes:

type Define el dato que obtenemos del campo de datos, por defecto es text, pero podemos poner: button,checkbox,color,date,datetime-local,email,file,hidden,image,month,number,password,radio,range,reset,
search,submit,tel,text,time,url,week

Atributo name es importante para la comunicación ya que el nombre del dato que se envía corresponde a lo que pongamos en este atributo.

Atributo value será el valor por defecto de un campo de entrada

Atributo pattern significa que no dejará enviar los datos y los marcará como erróneos si no cumple el patron o la expresión regular de este atributo. Tiene sentido en los campos de texto.
